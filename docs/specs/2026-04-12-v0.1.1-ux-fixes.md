## 1. Tutorial Deselection Bug

**Problem:** Clicking "Next" or "Get Started" in the welcome tutorial propagates to the document-level `pointerup` listener in `sceneClick.ts`. Since no body is under the cursor, the miss handler fires → `selection.set(null)` → the info panel closes. The tutorial then highlights an empty panel slot.

**Root cause:** The tutorial button click events propagate through to the scene click system. No `stopPropagation()` call on the button handlers.

**Fix:** Add `event.stopPropagation()` to both `next()` and `skip()` button click handlers in `WelcomeTutorial.svelte`.

**Files:**
- Modify: `src/lib/components/layout/WelcomeTutorial.svelte`

## 2. Mobile Bottom Drawer

**Problem:** On mobile, the info panel is `w-full` and covers the entire screen when a body is selected, blocking the 3D scene entirely.

**Solution:** On mobile (`< 640px`), replace the right-side slide-in panel with an Apple Maps-style bottom sheet drawer. Desktop behavior unchanged.

### Three snap points:
- **Peek** (~140px): drag handle, body name, subtitle, tracking badge. 3D scene mostly visible. This is the initial state on selection.
- **Half** (~50vh): scrollable facts, description, first sources. Default state when tapping a body.
- **Full** (~90vh): everything visible, full scroll.

### Interaction:
- **Drag handle:** small pill-shaped bar at the top of the drawer (standard iOS/Android pattern). Touch-drag up/down to resize.
- **Snap physics:** on drag release, snap to nearest snap point with spring easing. Velocity-aware — fast flick up from peek goes to full.
- **Swipe down from peek:** closes the drawer (deselects).
- **No backdrop dimming:** 3D scene stays visible and interactive behind the drawer.

### Implementation:
- New component: `src/lib/components/layout/MobileDrawer.svelte`
- Touch event handling (`touchstart`/`touchmove`/`touchend`) with `transform: translateY()` for 60fps drag
- `app/+page.svelte` conditionally renders `MobileDrawer` vs existing `<aside>` based on reactive viewport width check (`< 640px`)
- Drawer renders `<RightPanel />` inside — no content duplication

**Files:**
- Create: `src/lib/components/layout/MobileDrawer.svelte`
- Modify: `src/routes/app/+page.svelte`

## 3. Selected Object — Label Only

**Problem:** When a body is selected and zoomed in, too many labels are visible (selected + parent + children + all tier-eligible bodies). This clutters the view.

**Solution:** When a body is selected, show ONLY that body's label. When nothing is selected (`null` or `SOLAR_SYSTEM_VIEW`), the tier-based distance logic applies as before.

**Change in `computeVisibility()`:**
```typescript
if (sel !== null && sel !== SOLAR_SYSTEM_VIEW) {
  return sel === object.id;
}
```

Replaces the current block that also shows parent and children labels.

**Files:**
- Modify: `src/lib/components/scene/BodyLabel.svelte`

## 4. Planet Auto-Zoom Distance

**Problem:** Gas/ice giants have `cameraDistance` values that zoom too close, making the camera feel cramped on large planets.

**New values:**

| Planet | Current | New |
|---|---|---|
| Jupiter | 35 | 55 |
| Saturn | 30 | 60 |
| Uranus | 12 | 25 |
| Neptune | 12 | 25 |

Inner planets, Pluto, and all other bodies unchanged.

**Files:**
- Modify: `src/lib/registry/bodies/jupiter.ts`
- Modify: `src/lib/registry/bodies/saturn.ts`
- Modify: `src/lib/registry/bodies/uranus.ts`
- Modify: `src/lib/registry/bodies/neptune.ts`

## 5. Replace Starfield Background Texture

**Problem:** Current starfield texture is low-quality / generic.

**Solution:** Replace with the "Stars + Milky Way" panoramic equirectangular texture from [Solar System Scope](https://www.solarsystemscope.com/textures/) (CC BY 4.0). Same license as existing planet textures.

**Steps:**
1. Download the texture, convert to WebP, save as `static/textures/starfield_milkyway_8k.webp`
2. Update `Starfield.svelte` texture path to the new file
3. Tune `uBrightness` uniform (start at 2.0, adjust by eye)
4. Keep `BackgroundGradient.svelte` as-is
5. Delete old `static/textures/stars_milkyway_4k.webp`

No shader, geometry, or camera-follow changes.

**Files:**
- Modify: `src/lib/components/scene/Starfield.svelte`
- Create: `static/textures/starfield_milkyway_8k.webp`
- Delete: `static/textures/stars_milkyway_4k.webp`
